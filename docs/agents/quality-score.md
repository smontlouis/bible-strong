## Scoring Inputs

- Starts at 10.
- -2 when no colocated feature tests are found.
- -1 when no smoke path is mapped.
- -1 when the feature has no README.
- -1 when static scan finds many `any`, `TODO/FIXME`, console calls, or any `eslint-disable`.
- -1 for sensitive account/user-data surfaces.

## Agent Guidance

- Scores 8-10: normal validation path is usually enough.
- Scores 5-7: run broader checks and use the smoke checklist when touching user-facing behavior.
- Scores 1-4: inspect domain docs and sensitive-area policy before changing behavior; prefer adding tests or smoke coverage with the change.
